CNVD 2018 01084_D Link_DIR 645_reproduction

March 7, 2018 · 1 min · chiba

环境:

  1. x64 架构kali + qemu user mode
  2. mipsel 架构 debian

过程:

使用IDA 打开 cgibin，main函数里第一步是把argc[0]和各个cgi模块名进行cmp操作， 然后根据 cmp结果进入各个模块。

可以使用 -0 argv0 强制将目标的 argv[0] 设置为 argv0

cgi 脚本通常通过环境变量/标准输入 获取输入，为了单独运行CGI脚本， 需要构造 相对应的请求参数。

  1. # REQUEST_METHOD
  2. # cgibin_parse_request 中 获取以下参数:

CONTENT_TYPE CONTENT_LENGTH REQUEST_URL

CONTENT_LENGTH 基于POST 数据的长度 所以使用 $LEN 进行表示

可以看到请求为GET和POST请求都需要经过认证。

然后从环境变量中获取三个参数: EVENT，ACTION，SERVICE

QEMU可以通过 -E 设置环境变量 -0 设置参数

chroot . ./qemu-mipsel-static -0 "service.cgi" -E REQUEST_METHOD="POST" - E REQUEST_URI="service.cgi\?EVENT=CHECKFW%26ls%26" -E CONTENT_LENGTH=$LEN -E CONTENT_TYPE="application/x-www-form- urlencoded" -E HTTP_COOKIE="uid=123456" -E REMOTE_ADDR="127.0.0.1" -g 1234 -strace ./htdocs/cgibin

其中我们自己设置的 sess_ispoweruser 是经过不了校验的，不过能够在调试的时候手 动修改return value(v0)

我们将 EVENT参数设置为CHECKFW&ls& 传递到函数lxmldbc_system 中 system 执行的参数就被注入了命令。

添加 -strace 查看系统调用， 这里显示 exec format error是因为使用qemu user mode 之后不能够执行原本的execve，一种方式是使用 打过execve patch 的qemu-mipel-static版本。

另一种是在基于mips架构的debian系统中进行运行,不过IDA远程调试的时候某个指令 SIGTRAP了
